Arrays and ArrayLists

Definitions

Array - Block of consecutive memory locations of fixed size that hold values of same data type

ArrayList - Holds consecutive memory locations of variable size of values of Wrapper Classes, not built in

Capacity - Size of either array/ArrayList. Fixed for array, ArrayList capacity doubles every time it overflows

Index - Used to refer to array elements, first element in array/ArrayList has an index of 0 instead of 1

Element - Individual location in array/ArrayList

New Keyword - Used when declaring arrays/ArrayLists, make objects

Import Statements - Used to import Arraylist class, import.util.ArrayList and/or import.util.Collections


Arrays and ArrayLists

REMEMBER!!

1. Always import the java.util* library to use ArrayLists

2. Arrays and ArrayLists are not the same!! Arrays use Wrapper classes for their data types and therefore store objects, therefore methods for ArrayLists also reflect that (get methods, set methods, size() for ArrayList and length for array)


Impact and My Understanding

These data structures are very useful in java as they store a large sets of data under one variable name. This makes code more readable and doable, instead of naming and using a new unqiue variable each time. When the size of the data isn't known it can either be inputted by user or an ArrayList can be used. Using arrays and ArrayLists, it is also easy to segregate and sort data to do what we desire in one go in one place, which makes work more efficient.


For me, it was important to understand when to use arrays and ArrayLists respectfully. ArrayLists have more flexibility with their methods and are resizeable, while arrays are easier to read and use.
